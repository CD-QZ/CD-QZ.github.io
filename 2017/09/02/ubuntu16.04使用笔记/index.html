ubuntu16.04瞎折腾日志

开始用ubuntu原由

由于自己对计算机知识真的啥都不懂，想学编程，逛了n次论坛后心血来潮决定装个git用，然后又想用github和hexo搭建一个博客，在一步一步百度后总算装好了，然后纠结很久换来几个主题，因为没文化，在瞎弄的过程在突然网址打不开了，变成404,后来不知道怎么折腾的电脑输入法也没了，本来重装一个输入法就行了的，但是现在也想不出来怎么想的就想重装系统，而且是全部重装，不是只装c盘，觉得自己真的很神经，总之，再不知逛到什么论坛时就决定用linux了，然后百度各种新手适合用什么linux系统后终于选择ubuntu。

在装ubuntu的过程中也是百度各种方法，后来最纠结的部分就是分区的问题了，我是在知网上找到安装方法，根据别人的说法不用给很多空间，因此第一次就给50G，根目录5G，boot200M，swap2G，剩下全给home，结果装好后一直提示根目录空间不足，然后各种百度后据说删掉旧的内核，后来根据别人给的命令一阵瞎弄后，正常启动不了啦，变成测试模式，后来过啦很久后决定重装，几次过后决定给根目录40G，home50G，又单独挂载一个user给了50G（因为之前根目录空间都是被user占满啦，里面都是装的软件，之前以为这些都是因该在home里，而且大家都说根目录不用很大，空间都给home），在这过程中还有个问题是本来想把win7里的E盘（～200G）全用来装ubuntu的，结果压缩时只能压缩一半，所以从D盘由压缩50G空闲空间，还挺担心用的空间不再一个磁盘有没有影响， 如果将两个空间都挂home可以吗？现在是将D盘挂的user，好像没问题，这些问题都不懂，没有基础知识真是硬伤，以后好好补补。

ubuntu装好后顿时就喜欢上它啦，最然不会用，想在上面装git，hexo，写博客，可是。。。。git和nodejs都成功啦，hexo貌似也装好啦，因为有命令输./hexo–version后有内容，虽然不知到为什么是./hexo而不是直接hexo，可是输其他命令时，如初始化却显示没有hexo命令，折腾的太心累，最后还是先回win7装git，然后。。。总算顺利搭好啦。虽然估计以后还会有各种坑等着我。

2017.8.30

在通过清理ubuntu就内核镜像（因为boot占满啦）后，不小心输了个halt命令，然后就关机啦，然后死机了，然后重启，没网啦，wifi没有，不知道怎么弄，查了好久估计网卡驱动没了，话说之前也这样过一次，不敢随便清理内核镜像啦，然后又是重装，但这次不想完全初始化，想只初始化boot，这样下的东西还在，可是想给boot更多空间，没有空闲空间啦，（居然没想到从win上D盘再分点空间）选择从home分出500M挂载boot，在这样home就初始化了，原先boot的200M本打算给也给boot的，但是挂两个boot不知道会不会有问题，还是放弃了。

有人说不用给boot单独挂载，可是看见的安装说明是安装时要选择boot是挂载点，不懂！！！

####重新安装git
参考Ubuntu 16.04安装Git及GUI客户端 安装git以及GitKraken（看见别人说这个很酷炫，所以装来玩玩）。

装GitKraken时我把安装包放在软件盘，就是我win7上的D盘，好像ubuntu也可以看见win7上的磁盘，访问，但是win7不能访问。这样我可以尽量把东西装在win7上的磁盘里，可以节约空间。这样有问题吗？

疑惑: 但是装git时，我在D盘一个目录下运行终端，执行命令，想把git装这里，结果查看时还是装在/usr/bin/git /usr/share/man/man1/git.1.gz。

安装网易云音乐

我的手机和win7上都是装的网易云，很喜欢这个软件，因此想在ubuntu装一个同步，现在越来越觉得ubuntu可以满足我的一切需求，代替windows了。安装方法见这位大神 。看见一个博客介绍ubuntu一些基本软件安装方法，以后遇到不同安装包作为参考。

遇到问题：

  1. 官网安装.deb包后，直接点击安装没成功，不知道为什么，之前装qq也是？？？（还未解决）

  2. 用命令安装时，发生下列错误：

    dpkg: 处理归档 netease-cloud-music_0.9.0-2_amd64.deb (–install)时出错：
    无法访问归档文件: 没有那个文件或目录
    在处理时有错误发生：
    netease-cloud-music_0.9.0-2_amd64.deb

    开始以为是因为我装在win7的D盘，所以在该目录下执行终端命令,依旧无法访问，然后将安装包移到home下，当前目录执行和直接打开终端执行都不行。

    解决：原来是因为自己装的版本和命令里输入的版本不一致（都是太懒了，直接复制代码，被自己蠢哭了）。

2017.8.31

装vim和gcc

参考别人博客，虽然系统自带vi，还是装vim和gcc，感觉vim确实比vi好用。安装命令如下：

sudo apt-get install vim
sudo apt-get install g++

不知道为什么写的是g++，但是装的是gcc？？？对ubuntu命令还需继续学习，毕竟现在是什么都不懂的小白！！！！

什么是 Cmd Markdown

您可以使用很多工具书写 Markdown，但是 Cmd Markdown 是这个星球上我们已知的、最好的 Markdown 工具——没有之一 ：）因为深信文字的力量，所以我们和你一样，对流畅书写，分享思想和知识，以及阅读体验有极致的追求，我们把对于这些诉求的回应整合在 Cmd Markdown，并且一次，两次，三次，乃至无数次地提升这个工具的体验，最终将它演化成一个 编辑/发布/阅读 Markdown 的在线平台——您可以在任何地方，任何系统/设备上管理这里的文字。

1. 实时同步预览

我们将 Cmd Markdown 的主界面一分为二，左边为编辑区，右边为预览区，在编辑区的操作会实时地渲染到预览区方便查看最终的版面效果，并且如果你在其中一个区拖动滚动条，我们有一个巧妙的算法把另一个区的滚动条同步到等价的位置，超酷！

2. 编辑工具栏

也许您还是一个 Markdown 语法的新手，在您完全熟悉它之前，我们在 编辑区 的顶部放置了一个如下图所示的工具栏，您可以使用鼠标在工具栏上调整格式，不过我们仍旧鼓励你使用键盘标记格式，提高书写的流畅度。

3. 编辑模式

完全心无旁骛的方式编辑文字：点击 编辑工具栏 最右侧的拉伸按钮或者按下 Ctrl + M，将 Cmd Markdown 切换到独立的编辑模式，这是一个极度简洁的写作环境，所有可能会引起分心的元素都已经被挪除，超清爽！

4. 实时的云端文稿

为了保障数据安全，Cmd Markdown 会将您每一次击键的内容保存至云端，同时在 编辑工具栏 的最右侧提示 已保存 的字样。无需担心浏览器崩溃，机器掉电或者地震，海啸——在编辑的过程中随时关闭浏览器或者机器，下一次回到 Cmd Markdown 的时候继续写作。

5. 离线模式

在网络环境不稳定的情况下记录文字一样很安全！在您写作的时候，如果电脑突然失去网络连接，Cmd Markdown 会智能切换至离线模式，将您后续键入的文字保存在本地，直到网络恢复再将他们传送至云端，即使在网络恢复前关闭浏览器或者电脑，一样没有问题，等到下次开启 Cmd Markdown 的时候，她会提醒您将离线保存的文字传送至云端。简而言之，我们尽最大的努力保障您文字的安全。

通过管理工具栏可以：

新建：开始撰写一篇新的文稿

导出：将当前的文稿转化为 Markdown 文本或者 Html 格式，并导出到本地
列表：所有新增和过往的文稿都可以在这里查看、操作
模式：切换 普通/Vim/Emacs 编辑模式

7. 阅读工具栏

通过 预览区 右上角的 阅读工具栏，可以查看当前文稿的目录并增强阅读体验。

视图：互换左边编辑区和右边预览区的位置

阅读：心无旁骛的阅读模式提供超一流的阅读体验
全屏：简洁，简洁，再简洁，一个完全沉浸式的写作和阅读环境

8. 阅读模式

在 阅读工具栏 点击 或者按下 Ctrl+Alt+M 随即进入独立的阅读模式界面，我们在版面渲染上的每一个细节：字体，字号，行间距，前背景色都倾注了大量的时间，努力提升阅读的体验和品质。
